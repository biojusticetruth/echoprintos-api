About EchoprintOS

A lightweight, public timestamping layer for posts, images, and documents—designed to be read by humans, verified by machines, and trusted in court.

What it is

  • ECP ID — a human-readable Echoprint code (e.g., ECP-20251108105702814).
  • SHA-256 hash — a 64-hex fingerprint of the exact text or file bytes.
  • Timestamp (UTC) — when the record was echoprinted; formatted consistently for readability.
  • Optional links — a permalink to the original post and/or a media URL (image/file).

How it works

  1. Capture — For text, we hash the exact string. For files, we hash the exact bytes.
  2. Echoprint — We create a record: ECP ID, hash, and timestamp (UTC).
  3. Verify — Anyone can paste an ECP or hash into Verify and compare against the ledger entry.

Bitcoin anchoring (OpenTimestamps) is supported in the broader stack and can be exposed here when you’re ready.

Public feed policy (what you see)

  • Shows: ECP ID, UTC timestamp, title, and (when available) a link to source.
  • Hidden on cards: the raw hash (to reduce visual clutter) — still visible in the Verify view and in the downloadable certificate/JSON.

FAQ

What is an ECP ID?

A human-friendly identifier (e.g., ECP-YYYYMMDDHHMMSSmmm) that lets people reference a record without copying a 64-character hash.

Is the SHA-256 hash private?

No — a hash is a one-way fingerprint. It proves integrity (that content hasn’t changed) but does not reveal the content itself.

Where do timestamps come from?

The ledger stores an authoritative UTC timestamp at the moment of echoprinting. For Bitcoin anchoring, a proof (receipt) can be attached.

Can I verify by hash or by ECP?

Yes — the Verify tool accepts either a full 64-hex SHA-256 hash or an ECP ID.
